Phase 1 First Chat
1 — Install build tools
sudo dnf install -y cmake make gcc git gcc-c++ libcurl-devel
2 — Get files from USB
mkdir ~/chatbot && cd ~/chatbot
cp /run/media/$USER/DISK_IMG/* .
tar xzf llama.cpp-b7783.tar.gz
At home: git clone --branch b7783 --depth 1 https://github.com/ggerganov/llama.cpp
3 — Build
cd ~/chatbot/llama.cpp
mkdir build && cd build
cmake .. -DLLAMA_BUILD_EXAMPLES=ON
cmake --build . --config Release
⏲ 5–10 min build. Help a neighbor while you wait.
That’s all there is to it. An LLM is running on your local machine. This is all an LLM ever does! Text in, text out. Anything else we call “AI” is traditional software wrapped around that.
4 — Chat!
cd ~/chatbot/llama.cpp/build
./bin/llama-cli \
  -m ~/chatbot/microsoft_Phi-4-mini-instruct-Q4_K_M.gguf \
  -c 512 \
  -p "You are a helpful assistant."
You see a > prompt. Ask it something.
Type /exit to quit.
5 — One-shot mode (for scripts)
PROMPT="What is Fedora Linux?"

RESPONSE=$(
  LLAMA_LOG_VERBOSITY=1 ./bin/llama-completion \
    -m ~/chatbot/microsoft_Phi-4-mini-instruct-Q4_K_M.gguf \
    -p "$PROMPT" \
    -n 100 \
    -c 512 \
    -no-cnv \
    -r "<eor>" \
    --simple-io \
    --color off \
    --no-display-prompt
)

printf '%s\n' "$RESPONSE"
Prints the answer and exits. No interactive prompt.
This is how you call an LLM from a script.
